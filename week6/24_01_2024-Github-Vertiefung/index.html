Git Cheat Sheet

Was ist Git?

  • Git ist ein Versionskontrollsystem. Daten auf dem Rechner werden getrackt und im git gespeichert.
  • Tracking von Veränderungen & Dateiversionen
  • Kollaboration mit mehreren Teilnehmern möglich durch arbeiten in Branches & Merge requests

Basics

git init: Erstelle leere Git Repo

git clone [url]: Klone einre Remote Repo

git add .[file]: Dateien zum Staging für den Commit hinzufügen

git commit -m “Kommentar”: Committed alle vorgemerkten Dateien aus dem Staging mit Kommentar

git push: Pushe die Veränderungen zum Git

git log --oneline: Zeigt alle Commits der aktuellen Branch (komprimiert)

git status: Zeige alle geänderten Dateien

Branch

git branch: Übersicht aller Branches

git branch branchname: Neuen Branch erstellen

git checkout branchname: Wechsel zu einem anderen Branch & ins aktive Directory einchecken

git checkout -b branchname: Neuer Branch und direkt hinwechseln

git branch -d branchname: Branch löschen

Merge

  1. Wechsel zur Main Branch
  2. git merge branchname: Vom Hauptbranch aus 2 Branches mergen
  3. Änderungen Pushen
  4. git branch -d branchname : gemergten branch danach löschen
  5. Pushen nach dem Löschen

Code zurücksetzen

  1. git log --oneline: ID der commits identifizieren wo man zurückwill
  2. git checkout [commit_ID] .: Zurücksetzen auf den alten Stand mit commit ID
  3. git add .
  4. git commit -m "Kommentar, auf den alten stand zurückgesetzt"
  5. git push
